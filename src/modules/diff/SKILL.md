# Diff Module

## Overview
Generates file diffs by comparing current file contents against previous versions or snapshots.

## Role in System
A utility module that provides diff generation for code review, commit preparation, and change visualization. Used by the Git module for commit message generation and by the REPL for displaying changes.

## Dependencies
- **Depends on**: None (self-contained)
- **Used by**: Git module, REPL, potentially other modules that need change visualization
- **External deps**: Node.js built-in modules only (diff algorithm implementation)

## Key Services/Providers
| Service | Purpose |
|---|---|
| `DiffService` | Generates diffs between file versions. Provides the core diffing logic used by commit generators and code review tools. |

## Key Types/Interfaces
No dedicated types file — types are inferred or defined inline in the service.

## Coding Standards & Patterns
- **Single responsibility**: This module does one thing — generate diffs. No orchestration, no LLM calls.
- **No module dependencies**: Intentionally has zero imports from other Cast Code modules, making it easy to use anywhere without circular dependency risks.

## Business Rules
- Diffs are generated by comparing file contents (not by running `git diff`).
- Works with any file content, not just git-tracked files.

## Circular Dependencies
None. This is one of the few modules with zero dependencies on other Cast Code modules.

## Working on This Module
- **Extremely simple**: One service, one module file. Easy to understand and modify.
- **Adding diff formats**: If you need unified diff, side-by-side diff, or JSON diff output, extend `DiffService` with format options.
- **Integration point**: Most consumers call this service indirectly through the Git module or REPL commands.
