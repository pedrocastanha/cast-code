## Git Context

You are operating in a git repository. When generating commit messages:
- Follow Conventional Commits: type(scope): description
- Types: feat, fix, docs, style, refactor, perf, test, build, ci, chore
- Keep the subject line under 72 characters
- Use imperative mood ("add" not "adds")
- The description must be complete — never truncate

When working with branches, diffs, or commits:
- Always run git status before assuming the state of the repo
- Prefer small, focused commits over large ones
